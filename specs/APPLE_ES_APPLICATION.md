## 2. Product Overview

Sanctuary is a security runtime for AI coding agents running on macOS
workstations. The target agents include Claude Code, Cursor agents, Codex-style
CLIs, OpenAI Operator-style browser agents, Anthropic Computer Use-style agents,
and similar local automation systems.

Without Sanctuary, an agent launched by a developer inherits the user's local
privilege. It can read files in the home directory, execute shells, connect to
network services, and touch sensitive resources such as SSH keys, cloud
credentials, browser profiles, wallet extension storage, and password manager
state. Enterprises deploying agents to employee laptops need a way to scope,
approve, revoke, and audit agent behavior.

Sanctuary provides:

- **Capability scoping:** agents request typed capabilities such as
  `file.read`, `file.write`, `exec.shell`, and `net.connect` with explicit
  resource scope.
- **Human approval workflows:** sensitive grants surface through native local
  owner approval, including Touch ID where available.
- **Tamper-evident audit:** agent activity is recorded in a signed SHA-256
  hash-chained audit log.
- **Local containment:** out-of-scope operations by classified agents are
  detected in v0.1 and move to synchronous denial or invisibility after
  Endpoint Security approval.

Sanctuary v0.1 already ships the classifier, policy store, FSEvents detection,
CDP Guard, tamper-evident audit log, peer monitoring, `pf` revalidation, menu
bar console, and 410 tests. Endpoint Security is the missing Apple primitive
needed for the v0.2 enforcement layer.

## 3. Specific Endpoint Security Use Case

We request `com.apple.developer.endpoint-security.client` to subscribe to a
narrow set of Endpoint Security events generated by agent processes and by
processes attempting to tamper with Sanctuary.

### Events Requested

| ES event | Purpose |
| --- | --- |
| `ES_EVENT_TYPE_NOTIFY_EXEC` | Maintain the process identity index and classify fresh agent processes. |
| `ES_EVENT_TYPE_NOTIFY_FORK` | Track agent process hierarchies and inherited capability scope. |
| `ES_EVENT_TYPE_NOTIFY_EXIT` | Expire process identity and session cache entries. |
| `ES_EVENT_TYPE_AUTH_EXEC` | Mediate high-risk process launches by agents, including restricted shell execution. |
| `ES_EVENT_TYPE_AUTH_OPEN` | Mediate file reads and writes against capability grants and protected resources. |
| `ES_EVENT_TYPE_AUTH_CREATE` | Enforce workspace-scoped write grants for file creation. |
| `ES_EVENT_TYPE_AUTH_RENAME` | Prevent out-of-scope mutation or tamper through rename operations. |
| `ES_EVENT_TYPE_AUTH_UNLINK` | Prevent deletion of protected resources, policy DB files, and audit logs. |
| `ES_EVENT_TYPE_AUTH_SIGNAL` | Prevent unauthorized termination of `sanctuaryd` where policy permits. |
| `ES_EVENT_TYPE_NOTIFY_MOUNT` | Detect mount changes that affect protected resource visibility or audit integrity. |

We do not request access to keystrokes, screen contents, audio, camera streams,
or file contents. Sanctuary uses Endpoint Security for process, file, and
tamper mediation only.

### Why Endpoint Security Is Necessary

We have implemented lower-privilege alternatives and found them insufficient
for enforcement:

- **FSEvents** is post-hoc. It can record that an agent touched a path, but the
  read has already completed before Sanctuary receives the event.
- **`pf`** is useful for one narrow loopback network surface: Chrome DevTools
  Protocol. It cannot mediate filesystem or process events.
- **Network Extension** is not a filesystem or process mediation API.
- **App Sandbox** only applies to apps that opt into it. Sanctuary must mediate
  third-party agents that users install independently.
- **Wrapper-based approaches** are advisory. Agents can bypass wrappers through
  child shells, direct syscalls, or embedded runtimes.

Endpoint Security is the Apple-provided API that gives Sanctuary synchronous
authorization decisions for file and process operations. That is required for
the product's v0.2 capability runtime.

## 4. Target Customers and Deployment Scenarios

Sanctuary is designed for:

1. **Individual developer workstations** running local AI coding agents beside
   SSH keys, cloud credentials, browser profiles, and wallet state.
2. **Engineering team fleets** deploying agents to developer laptops and
   requiring central policy, revocation, and audit.
3. **Regulated organizations** deploying AI agents under SOC 2, ISO 27001,
   HIPAA, financial-services, or government-contractor controls.

The enterprise question Sanctuary answers is:

```text
What did the agent touch, was it approved, can we revoke it, and can we prove it?
```

## 6. Privacy and Anti-Abuse Commitments

Sanctuary is intentionally local-first:

- Raw Endpoint Security event streams are processed locally.
- Enforcement decisions do not require a cloud round trip.
- The runtime does not phone home by default.
- Audit logs are local unless the user or fleet admin explicitly enables export
  through a future Hardener Cloud policy.
- Process identity collection stores environment variable names only, never
  values.
- Sanctuary never reads file contents to make policy decisions.
- The runtime does not capture keystrokes, screen contents, audio, or camera
  streams.
- The full runtime source is published under AGPL v3.

These commitments are part of the public threat model and coverage gap
documents in the repository.

## 7. Why This Entitlement Should Be Approved

1. **The use case is a security use case.** Sanctuary mediates third-party
   process behavior on endpoints, similar in shape to EDR and DLP products, but
   focused on AI agent containment.
2. **The scope is narrow.** The requested events are limited to process/file
   mediation and runtime self-defense.
3. **The product addresses a real enterprise need.** Companies deploying AI
   coding agents need local capability scoping, approval, revocation, and audit
   on developer laptops.
4. **The implementation is inspectable.** The runtime is AGPL v3 and public on
   GitHub. Apple, customers, and researchers can inspect event handling.
5. **The product strengthens the macOS ecosystem.** It gives enterprises a path
   to deploy AI agents on macOS while preserving the security expectations of
   managed developer workstations.
