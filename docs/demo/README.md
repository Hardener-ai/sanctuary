# Sanctuary Launch Demo Assets

This directory contains launch-facing visual assets for Sanctuary by Hardener.

## Files

- `screenshots/01-hero.png` - opening title frame
- `screenshots/02-menu-bar-protections.png` - menu bar protection overview
- `screenshots/03-cdp-guard-block.png` - CDP Guard block visualization
- `screenshots/04-audit-feed.png` - audit feed and hash-chain verification
- `screenshots/05-e2e-proof.png` - reproducible test/e2e proof frame
- `screenshots/06-roadmap.png` - platform roadmap frame
- `video/sanctuary-demo.mp4` - 24-second silent launch demo cut

## Source

The frames are generated by:

```sh
swift scripts/render_marketing_assets.swift
ffmpeg -y -f concat -safe 0 -i docs/demo/video/demo-concat.txt \
  -vf fps=30,format=yuv420p -movflags +faststart \
  docs/demo/video/sanctuary-demo.mp4
```

The renderer uses the Hardener brand assets in `docs/assets/brand/`.

## Accuracy Notes

These are branded launch/demo frames, not raw macOS screen captures. They are
based on the current product surface and live e2e evidence:

- `swift test`: 376 tests passing
- `./e2e/run-all.sh`: 6 PASS / 2 SKIP without `E2E_PF=1`
- CDP Guard is the v0.1 real-time block
- Filesystem and extension storage are detection-only in v0.1
- v0.2 roadmap depends on Apple's Endpoint Security entitlement

Use these in README, issue posts, short social clips, and launch planning.
For App Store-style screenshots or press kits, capture real macOS UI once the
Developer ID signed build is ready.
